# 小程序编译问题修复说明

## 问题描述

微信开发者工具报错：
```
getAppConfig error with backend: app.json: app.json is not found in the project root directory
```

## 问题原因

在 uni-app 中，**tabBar 的第一个页面必须是 `pages` 数组的第一个页面**。这是微信小程序的强制要求。

### 错误的配置

```json
{
  "pages": [
    {
      "path": "pages/home/home",  // ❌ 第一个页面不是 tabBar 页面
      ...
    },
    {
      "path": "pages/camp/camp",  // tabBar 第一个页面
      ...
    }
  ],
  "tabBar": {
    "list": [
      {
        "pagePath": "pages/camp/camp"  // tabBar 第一个页面
      }
    ]
  }
}
```

### 正确的配置

```json
{
  "pages": [
    {
      "path": "pages/camp/camp",  // ✅ tabBar 第一个页面必须是 pages 数组第一个
      ...
    },
    {
      "path": "pages/home/home",
      ...
    }
  ],
  "tabBar": {
    "list": [
      {
        "pagePath": "pages/camp/camp"  // tabBar 第一个页面
      }
    ]
  }
}
```

## 修复内容

1. ✅ 调整了 `pages.json` 中 `pages` 数组的顺序
2. ✅ 将 tabBar 的第一个页面（`pages/camp/camp`）移到 `pages` 数组的第一位
3. ✅ 确保所有 tabBar 页面都在 `pages` 数组中

## 验证步骤

1. 在 HBuilderX 中重新编译项目
2. 检查 `unpackage/dist/dev/mp-weixin/` 目录是否生成了 `app.json`
3. 在微信开发者工具中打开编译后的项目，确认不再报错

## 注意事项

- **tabBar 页面顺序**：tabBar 中列出的所有页面都必须在 `pages` 数组中，且第一个 tabBar 页面必须是 `pages` 数组的第一个页面
- **页面路径**：确保所有页面路径都正确，且对应的 `.vue` 文件存在
- **编译输出**：编译后的文件应该在 `unpackage/dist/dev/mp-weixin/` 目录下

## 相关文件

- `frontend/pages.json` - 页面路由配置文件
- `frontend/manifest.json` - 小程序配置文件
- `frontend/project.config.json` - 微信开发者工具项目配置
